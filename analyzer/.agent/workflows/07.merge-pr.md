# PR Review & Merge to Staging

This workflow defines the safe, recommended process for reviewing an external Pull Request (e.g., from a fork or teammate), testing it, and merging it into the `staging` branch.

## 1. Review and Analysis
- Ask the user for the PR number if not provided.
- Read the PR title and description to understand the scope.
- Use `mcp_github_get_pull_request_files` to list changed files.
- Analyze the changes for logic flaws, security vulnerabilities, or style violations.
- Use `mcp_github_get_pull_request_status` to ensure CI/CD and automated checks are passing.
- Summarize the findings for the user. If issues exist, wait for them to be addressed or use `mcp_github_create_pull_request_review` to leave comments.

## 2. Target Branch Verification
- All feature additions or bug fixes ought to be merged into `staging` first.
- Check the PR's `base` branch. If it targets `main`, inform the user. The PR target should ideally be changed to `staging` before proceeding (this can be done on GitHub).

## 3. Local Testing (Optional)
If local testing is requested or required due to complex changes:
1. Fetch the PR into a temporary local branch:
   ```bash
   git fetch origin pull/<PR_NUMBER>/head:pr-<PR_NUMBER>
   git checkout pr-<PR_NUMBER>
   ```
2. Ask the user to verify the app's behavior locally (e.g., run `npm run dev`).
3. Wait for the user's explicit confirmation before proceeding to merge.

## 4. Merge to Staging
Once approved:
- Use **Squash and Merge** (the recommended strategy to keep history clean).
- Use `mcp_github_merge_pull_request` with `merge_method: "squash"` to perform the merge via GitHub API.
- Create a clear, concise squash commit title and message.

## 5. Local Environment Sync
After the PR is merged on GitHub, sync the local environment:
```bash
// turbo
git checkout staging
// turbo
git pull origin staging
```

If a temporary local branch was created during Step 3, clean it up:
```bash
// turbo
git branch -D pr-<PR_NUMBER>
```

## 6. Completion
- Confirm to the user that the PR has been successfully squashed into `staging`.
- Provide a summary of what was merged and ask if there are further tasks.
